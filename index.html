R3A Secure Language Engine

A high-performance, local-first cryptographic runtime designed for tamper-proof data handling and secure string manipulation.


R3A (Reliable 3-layer Architecture) is an open-standard encryption engine that operates entirely within the client-side environment. By utilizing a Zero-Knowledge approach, R3A ensures that cryptographic keys and plaintext are never transmitted over the network, effectively neutralizing server-side vulnerabilities and data breaches.

Technical Architecture

The engine follows a strict three-phase execution pipeline to ensure data integrity:

1. Shifting Layer

Each character in the input string undergoes a modular shift based on the derived index of the secret key. This prevents simple frequency analysis attacks.

2. Integrity Layer

A 36-bit checksum is generated using a non-linear hashing algorithm. This checksum is appended to the payload to detect any manual tampering of the encrypted code.

3. Protocol Layer

Data is wrapped in a versioned R3A envelope. This allows the engine to maintain backward compatibility while upgrading security protocols in future versions.

4. Local Persistence

Designed for offline-first applications, R3A can be deployed in environments with zero internet connectivity while maintaining full cryptographic strength.

Security Standards

R3A implements a defensive coding pattern to prevent common web-based attacks:

  • Brute Force Mitigation: Built-in key strength meter encourages high-entropy secrets.
  • XSS Protection: Output is sanitized and Base64 encoded to prevent script injection in stored data.
  • Memory Safety: Variables are cleared after execution to minimize the risk of memory heap snapshots.

R3A Documentation

1. Protocol Format

Versioned structure ensures valid decryption and prevents tampering.


R3A|v1|checksum|payload

R3A = signature, v1 = version, checksum = integrity, payload = Base64 encrypted.

2. Hash Function

Generates a hash for key derivation and integrity check.


function hash(str){
  let h = 0;
  for(let i=0; i<str.length; i++){
    h = (h << 5) - h + str.charCodeAt(i);
  }
  return Math.abs(h).toString(36);
}

3. Encryption Logic

Shifts each character with the secret key and Base64 encodes.


function mix(text, key){
  let r = '';
  for(let i=0; i<text.length; i++){
    r += String.fromCharCode(
      text.charCodeAt(i) +
      (key.charCodeAt(i % key.length) % 17)
    );
  }
  return btoa(r);
}

4. Decryption Logic


function unmix(data, key){
  let d = atob(data),
      r = '';
  for(let i=0; i<d.length; i++){
    r += String.fromCharCode(
      d.charCodeAt(i) -
      (key.charCodeAt(i % key.length) % 17)
    );
  }
  return r;
}

Instead of relying heavily on third-party services, I try to build things locally to understand how they actually work.

R3A Engine

The R3A Engine is my main cryptography experiment and the core project under R3ACT. It’s a layered encryption runtime designed to be simple, portable, and dependency-free. The engine uses a versioned signature format (R3A|v1) so the encryption logic can evolve over time without breaking older encrypted data.

Internally, the engine combines a custom character-shifting algorithm with a Base36 checksum system. This allows the engine to detect even tiny changes in encrypted data. If the data is altered, the signature fails validation, and the engine immediately rejects it. The goal isn’t to replace professional cryptography standards, but to deeply understand how encryption systems are designed and validated.
